# Contributing

Thank you for considering a contribution. This project follows a small but strict contribution model so the repository remains reproducible, testable, and easy to index.

## Development Workflow

1. Create a branch for your change.
2. Update code, documentation, or tests together.
3. Run the project test suite.
4. Open a pull request with a clear summary and verification notes.

## Quality Expectations

Changes should preserve the repository's SEO structure, data-source notes, and Resource Portals section. When adding features, include tests or examples whenever possible.

## Documentation Expectations

If you add a new data source, API endpoint, or keyword cluster, update `docs/api-integration.md`, `docs/seo-keywords.md`, and `docs/traffic-page-matrix.md` accordingly.
